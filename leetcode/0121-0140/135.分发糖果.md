# 题目描述

这是LeetCode上的[135.分发糖果](https://leetcode.cn/problems/candy/)。

难度：**困难**

Tag：贪心、数组

`n`个孩子站成一排。给你一个整数数组`ratings`表示每个孩子的评分。

你需要按照以下要求，给这些孩子分发糖果：

- 每个孩子至少分配到`1`个糖果。
- 相邻两个孩子中，评分更高的那个会获得更多的糖果。

请你给每个孩子分发糖果，计算并返回需要准备的**最少糖果数目**。

示例1：

```
输入：ratings = [1,0,2]
输出：5
解释：你可以分别给第一个、第二个、第三个孩子分发 2、1、2 颗糖果。
```


示例2：

```
输入：ratings = [1,2,2]
输出：4
解释：你可以分别给第一个、第二个、第三个孩子分发 1、2、1 颗糖果。
     第三个孩子只得到 1 颗糖果，这满足题面中的两个条件。
```

提示：

- `n == ratings.length`
- `1 <= n <= 2 * 10000`
- `0 <= ratings[i] <= 2 * 10000`

# 解题思路

## 方法一：两次遍历

规则定义：设学生`A`和学生`B`左右相邻，`A`在`B`左边

- **左规则**：当`ratings B > ratings A`时，`B`的糖比`A`的糖数量多。
- **右规则**：当`ratings A > ratings B`时，`A`的糖比`B`的糖数量多。

相邻的学生中，评分高的学生必须获得更多的糖果**等价于**所有学生满足左规则且满足右规则。

左规则用来**从左往右遍历**的规则，有规则是用来**从右往左遍历**的规则。

### 算法流程

- 先从左至右遍历学生成绩`ratings`，按照以下规则给糖，并记录在`left`中：
  - 先给所有学生`1`颗糖；
  - 若`ratings[i] > ratings[i−1]`，则第`i`名学生糖比第`i−1`名学生多`1`个。
  - 若`ratings[i] <= ratings[i−1]`，则第`i`名学生糖数量不变。（交由从右向左遍历时处理。）
  - 经过此规则分配后，可以保证所有学生糖数量**满足左规则**。
- 同理，在此规则下从右至左遍历学生成绩并记录在`right`中，可以保证所有学生糖数量**满足右规则**。
- 最终，取以上2轮遍历`left`和`right`对应学生糖果数的**最大值**，这样则**同时满足左规则和右规则**，即得到每个同学的最少糖果数量。

### 正确性

**证明：取`max(left[i], right[i])`能同时满足左规则和右规则**
- 即令`candies[i] = max(left[i], right[i])`，其中：
- `left[i]`由左规则递推得到：若`ratings[i] > ratings[i-1]`，则`left[i] = left[i-1] + 1`，否则`left[i] = 1`。
- `right[i]`由右规则递推得到：若`ratings[i] > ratings[i+1]`，则`right[i] = right[i+1] + 1`，否则`right[i] = 1`。

**先证明满足左规则：若`ratings[i] > ratings[i-1]`，则`candies[i] > candies[i-1]`**

- 对于位置`i`，由于`ratings[i] > ratings[i-1]`，得到`left[i] = left[i-1] + 1`，故`left[i] > left[i-1]`。
- 对于位置`i - 1`，由于`ratings[i-1] < ratings[i]`，得到`right[i-1]`未被更新，保持数值1。
- 于是`candies[i-1] = max(left[i-1], 1) = left[i-1]`（因为`left[i-1] ≥ 1`）。
- 而`candies[i] ≥ left[i] = left[i-1] + 1 > left[i-1] = candies[i-1]`。
- 因此`candies[i] > candies[i-1]`，左规则成立。

**再证明满足左规则：若 `ratings[i] > ratings[i+1]`，则 `candies[i] > candies[i+1]`**

- 对于位置`i`，由于`ratings[i] > ratings[i+1]`，得到`right[i] = right[i+1] + 1`，故`right[i] > right[i+1]。
- 对于位置`i + 1`，由于`ratings[i+1] < ratingsp[i]`，得到`left[i+1]`未被更新，保持数值1。
- 于是`candies[i+1] = max(right[i+1], 1) = right[i+1]`
- 而`candies[i] ≥ right[i] = right[i+1] + 1 > right[i+1] = candies[i+1]`。
- 因此 `candies[i] > candies[i+1]`，右规则成立。

### 代码

```c++
class Solution {
public:
    int candy(vector<int>& ratings) {
        int n = ratings.size();
        // left[i]: 仅考虑左规则时，第 i 个孩子至少需要的糖果数
        vector<int> left(n, 1);
        // right[i]: 仅考虑右规则时，第 i 个孩子至少需要的糖果数
        vector<int> right(n, 1);

        // 左规则：从左向右遍历，若右边评分高，则右边糖果比左边多
        for (int i = 1; i < n; ++i) {
            if (ratings[i] > ratings[i - 1])
                left[i] = left[i - 1] + 1;
        }

        // 先初始化 count 为最右边孩子的糖果数（左规则已确定）
        int count = left[n - 1];

        // 右规则：从右向左遍历，若左边评分高，则左边糖果比右边多
        for (int i = n - 2; i >= 0; --i) {
            if (ratings[i] > ratings[i + 1])
                right[i] = right[i + 1] + 1;
            // 取 left[i] 和 right[i] 的最大值，同时满足两个规则
            count += max(left[i], right[i]);
        }

        return count;
    }
};
```

### 复杂度分析

- **时间复杂度**$O(N)$：遍历两遍数组即可得到结果；
- **空间复杂度**$O(N)$：需要借用`left`，`right`的线性额外空间。



## 方法二：常数空间遍历

从左往右遍历，当前小孩与前一个小孩的关系只有相等、大于、小于。遍历从第二个小孩开始。

当遍历所有评分，当前小孩与前一个小孩的评分没有小于关系时：

- 当前小孩的评分大于前一个小孩的评分时，当前小孩的糖果数等于前一个小孩的糖果数加`1`。
- 否则，则当前小孩的糖果数设置为`1`

```
评分相等在最后，输出17
                    *
                *   *
            *   *   *
        *   *   *   *
    *   *   *   *   *   *   *
    1   3   5   7   9   9   9

评分相等在最前，输出17
                            *
                        *   *
                    *   *   *
                *   *   *   *
    *   *   *   *   *   *   *
    1   1   1   3   5   7   9

评分相等中间，输出13
            *               *
        *   *           *   *
    *   *   *   *   *   *   *
    1   3   5   5   5   7   9
```

当遍历评分，当前小孩评分小于前一个小孩评分，只出现`1`次时，处理逻辑与不出现一样，只是单调递增变成了多次递增。

```
输出16
                * |
            *   * |         *
        *   *   * |     *   *
    *   *   *   * | *   *   *
    1   3   5   7 | 6   8   9
```

当遍历评分，当前小孩评分小于前一个小孩评分出现多次，但不超过最近大于出现的次数时：

- 当大于时，当前小孩糖果数等于前一个小孩糖果数目加1
- 当等于时，糖果数目为1
- 当小于时，增加的糖果数目，是为到目前为止小于出现的次数。

```
输出15
                * |   |
            *   * |   |
        *   *   * | + |     *
    *   *   *   * | * | +   *
    1   3   5   7 | 6 | 5   6
```

当遍历评分，当前小孩评分小于前一个小孩评分出现多次，且等于最近大于出现的次数时：

- 处理流程与上一种情况相同。此时，7可以算作递增的长度。

```
输出16
                * |
            *   * | @
        *   *   * | +   @
    *   *   *   * | *   +   @
    1   3   5   7 | 6   5   4
```

当遍历评分，当前小孩评分小于前一个小孩评分出现多次，且超过最近大于出现的次数时：

- 当大于时，当前小孩糖果数等于前一个小孩糖果数目加1
- 当等于时，糖果数目为1
- 当小于时，增加的糖果数目，是为到目前递减元素的个数。
  - 如果递增序列元素个数，等于递减序列元素个数，需要序列长度增加1。
  - 若当前数值为3，
    - 递减序列长度为4，分别是6，5，4，3
    - 递增序列长度为4，分别是1，3，5，7
    - 此时需要增加1，因为7在此时被划分到递减序列的元素

```
输出27
              | #
              | &   #
              | *   &   #
            * | *   @   &   #
        *   * | *   +   @   &   #
    *   *   * | *   *   +   @   &   #
    1   3   5 | 7   6   5   4   3   2
```

### 算法流程

 **初始化**：

- `pre`：前一个小朋友糖果数目，由于第一个小朋友糖果为1，且判断从第二个小朋友开始，故初始化为`1`
- `inc`和`dec`：分别统计递增与递减序列元素的个数，`inc`初始化为`1`，`dec`初始化为0
- `res`：糖果总数，第一个小朋友一定有一个，故初始化为`1`。

**循环遍历所有评分，从第二个开始**：

- `ratings[i] > ratings[i - 1]`时
  - `dec = 0`
  - `pre += 1`：当前元素的糖果个数，等于前一个元素糖果个数加1。
  - `res += pre`
  - `inc = pre`：更新序列长度，在递增时，增加的糖果个数与递增元素个数一样。
- `ratings[i] < ratings[i - 1]`时
  - `dec++`
  - 当`dec`等于`inc`时，递增元素序列的最后一个元素需要多给1个糖果才能复合要求。
  - `res += dec`
  - `pre = 1`：由于当前元素可能成为递增序列的第一个元素，所以准备`pre`的数值是1个糖果
- `ratings[i] == ratings[i - 1]`时
  - `dec = 0`：重置递减序列长度
  - `inc = 1`，递增序列从自己这个元素开始算。
  - `res += 1`：增加糖果总数
  - `pre = 1`：充值递增序列前一个元素的糖果数目。

### 代码

```c++
class Solution {
public:
    /**
     * 分发糖果（常数空间一次遍历解法）
     * 思路：从左到右遍历，动态维护递增序列长度 inc 和递减序列长度 dec，
     *       当递减长度等于递增长度时，波峰需要增高1颗糖。
     * 
     * @param ratings 每个孩子的评分数组
     * @return 最少需要分配的糖果总数
     */
    int candy(vector<int>& ratings) {
        // res: 糖果总数，初始化为第一个孩子的1颗
        int res = 1;
        // pre: 前一个孩子分到的糖果数，初始为1
        int pre = 1;
        // inc: 当前递增序列的长度（也是最后一个递增孩子的糖果数）
        int inc = 1;
        // dec: 当前递减序列的长度（从波峰下降的步数）
        int dec = 0;

        // 从第二个孩子开始遍历
        for (int i = 1; i < ratings.size(); ++i) {
            if (ratings[i] > ratings[i - 1]) {
                // 情况1：递增（右边评分更高）
                dec = 0;                     // 重置递减计数器
                pre = pre + 1;               // 当前孩子糖果数 = 前一个 + 1
                res += pre;                  // 累加到总数
                inc = pre;                   // 更新递增序列长度（即当前波峰高度）
            } 
            else if (ratings[i] < ratings[i - 1]) {
                // 情况2：递减（右边评分更低）
                dec++;                       // 递减深度增加1
                // 如果递减长度等于递增长度，说明波峰需要增高1颗糖
                // 否则递减序列的第一个孩子会与波峰相等，违反规则
                if (dec == inc) {
                    dec++;                   // 波峰增高，通过 dec++ 补偿
                }
                res += dec;                  // 累加当前递减孩子对应的增量
                pre = 1;                     // 重置 pre，为下一段递增做准备
            } 
            else {
                // 情况3：评分相等
                dec = 0;                     // 重置递减计数器
                inc = 1;                     // 新递增序列从此开始，长度为1
                res += 1;                    // 当前孩子只需1颗糖
                pre = 1;                     // 重置 pre
            }
        }
        return res;                          // 返回总糖果数
    }
};
```

### 复杂度分析

- **时间复杂度**$O(N)$：其中`N`是孩子的数量。
- **空间复杂度**$O(1)$：我们只需要常数的空间保存若干变量。
