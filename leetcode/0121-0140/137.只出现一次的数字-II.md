# 题目描述

这是LeetCode上的[137.只出现一次的数字-II](https://leetcode.cn/problems/single-number-ii/)。

难度：**中等**

Tag：位运算、数组

给你一个整数数组`nums`，除某个元素仅出现**一次**外，其余每个元素都恰出现**三次。**请你找出并返回那个只出现了一次的元素。

你必须设计并实现线性时间复杂度的算法且使用常数级空间来解决此问题。

示例1：

```
输入：nums = [2,2,3,2]
输出：3
```


示例2：

```
输入：nums = [0,1,0,1,0,1,99]
输出：99
```

提示：

- `1 <= nums.length <= 3 * 10000`
- `-2^{31} <= nums[i] <= 2^{31} - 1`
- `nums`中，除某个元素仅出现**一次**外，其余每个元素都恰出现**三次**

# 解题思路

考虑数字的二进制形式，对于出现三次的数字，各**二进制位**出现的次数都是`3`的倍数。因此，统计所有数字的各二进制位中`1`的出现次数，并对`3`求余，结果则为只出现一次的数字。

```
                nums= [3,5,3,3]
                    3 = 0 0 1 1
                    3 = 0 0 1 1
                    3 = 0 0 1 1
                    5 = 0 1 0 1
二进制各位中 1 的个数      0 1 3 4
          对 3 求余      0 1 0 1 = 5
```

## 方法一：有限状态自动机

**各二进制位的位运算规则相同，因此只需考虑一位即可**。

对于所有数字中的某二进制位`1`的个数，存在`3`种状态，即对`3`余数为`0`，`1`，`2`。

- 若输入二进制位`0`，则状态不变。
- 若输入二进制位`1`，则转至下一个状态，即按照`0,1,2,0,1,2,0,...`顺序转换；

```
        0             0             0
      +---+         +---+         +---+
      ↓   ↑         ↓   ↑         ↓   ↑
      -----   1     -----    1    -----
      | 0 |-------->| 1 |-------->| 2 |
      -----         -----         -----
        ↑                           ↓
        +---------------------------+
                     1
```

由于二进制只能表示`0`，`1`，因此需要使用两个二进制位来表示`3`个状态。设此两位分别为`two`，`one`，则状态转换变为：`00,01,10,00,...`

```
        0              0              0
      +----+         +----+         +----+
      ↓    ↑         ↓    ↑         ↓    ↑
      ------    1    ------    1    ------
      | 00 |-------->| 01 |-------->| 10 |
      ------         ------         ------
        ↑                             ↓
        +-----------------------------+
                     1
```

需要通过**状态转换表**导出**状态转换的计算公式**。首先回忆一下位运算特点，对于任意二进制位`x`，有：

- 异或运算：`x ^ 0 = x` ， `x ^ 1 = ~x`
- 与运算：`x & 0 = 0` ， `x & 1 = x`

**计算`one`比特的方法**：

设当前状态为`two one`，此时输入二进制位`n`。通过对状态表的情况拆分，可推出`one`的计算方法为：

```c
if two == 0:
  if n == 0:
    one = one
  if n == 1:
    one = ~one
if two == 1:
    one = 0
```

引入**异或运算**，可将以上拆分简化为：

```c
if two == 0:
    one = one ^ n
if two == 1:
    one = 0
```

引入**与运算**，可继续简化为：

```c++
one = one ^ n & ~two
```

计算`two`方法：

由于是先计算`one`，因此应在新`one`的基础上计算`two`。

```
计算 one 前的状态转换
    00------>10------>10
    ↑                  |
    +------------------+
计算 one 后的状态转换图
    01------>00------>10
    ↑                  |
    +------------------+
将 one 和 two 对调位置
    10------>00------>01
    ↑                  |
    +------------------+
调整各个状态位置画法
    00------>10------>10
    ↑                  |
    +------------------+
若使用计算后的新one，则可以通过相同的公式计算出two
```

修改为新`one`后，得到了新的状态图。观察发现，可以使用同样的方法计算`two`，即：

```c++
two = two ^ n & ~one
```

**返回值**：

以上是对数字的二进制中“一位”的分析，而`int`类型的其他`31`位具有相同的运算规则，因此可将以上公式直接套用在`32`位数上。

遍历完所有数字后，各二进制位都处于状态`00`和状态`01`（取决于“只出现一次的数字”的各二进制位是`1`还是`0`），而此两状态是由`one`来记录的（此两状态下`twos`恒为`0`），因此返回`ones`即可。

### 代码

```c++
class Solution {
public:
    /**
     * 找出数组中只出现一次的数字（其他数字均出现三次）
     * 
     * 思路：使用位运算的有限状态机，模拟三进制计数器。
     * 对于每一个二进制位，统计该位出现 1 的次数模 3。
     * 利用两个变量 ones 和 twos 记录状态：
     *   - (twos, ones) = 00 表示该位出现 0 次或 3 次
     *   - (twos, ones) = 01 表示该位出现 1 次
     *   - (twos, ones) = 10 表示该位出现 2 次
     * @param nums 输入数组
     * @return 只出现一次的数字
     */
    int singleNumber(vector<int>& nums) {
        int ones = 0, twos = 0;
        for (int num : nums) {
            // 更新 ones：当前位出现次数为奇数时，ones 置 1
            // 公式：ones = (ones ^ num) & ~twos 的标准写法
            ones = (ones ^ num) & ~twos;
            // 更新 twos：当前位出现次数为 2 时，twos 置 1
            twos = (twos ^ num) & ~ones;
        }
        // 最终只出现一次的数字存储在 ones 中
        return ones;
    }
};
```

### 复杂度分析

- **时间复杂度**$O(N)$：其中`N`位数组`nums`的长度；遍历数组占用 $O(N)$，每轮中的常数个位运算操作占用 $O(32×3×2)=O(1)$。
- **空间复杂度**$O(1)$：变量`ones`，`twos`使用常数大小的额外空间。



## 方法二：遍历统计

> 此方法相对容易理解，但效率较低，总体推荐方法一。

使用**与运算**，可获取二进制数字`num`的最右一位：`n_0 = num & 1`

配合无符号右移操作，可获取`num`所有位的值：`num = num >> 1`

建立一个长度为`32`的数组`counts`，通过以上方法可记录所有数字的各二进制位的`1`的出现次数。

将`counts`各元素对`3`求余，则结果为“只出现一次的数字”的各二进制位。

利用**左移操作**和**或运算**，可将`counts`数组中各二进位的值恢复到数字`res`上

最终返回`res`即可。

### 代码

实际上，只需要修改求余数值`m`，即可实现解决**除了一个数字以外，其余数字都出现`m`次**的通用问题。

```c++
class Solution {
public:
    int singleNumber(vector<int>& nums) {
        // 创建一个长度为 32 的计数数组，用于统计每个位上 1 出现的次数
        int counts[32] = {0};

        for (int num : nums) {
            // 将 num 转换为无符号整数，以确保右移时高位补 0
            unsigned int unum = static_cast<unsigned int>(num);
            for (int j = 0; j < 32; ++j) {
                // 统计当前最低位的值（0 或 1），累加到 counts[j]
                counts[j] += (unum & 1);
                // 无符号右移一位，继续处理下一位
                unum >>= 1;
            }
        }

        // 从 counts 数组中恢复结果
        int res = 0;
        int m = 3;  // 模数，因为其他数字出现三次
        for (int i = 0; i < 32; ++i) {
            // 先将结果左移一位，为下一位腾出空间
            res <<= 1;
            // 取 counts[31 - i] 对 m 的余数，作为当前最高位的值
            // 注意：Java 代码中是从高位向低位构建，所以用 counts[31 - i]
            res |= (counts[31 - i] % m);
        }
        return res;
    }
};
```

### 复杂度分析

- **时间复杂度**$O(N)$：其中`N`位数组`nums`的长度；遍历数组占用 $O(N)$，每轮中的常数个位运算操作占用 $O(1)$。
- **空间复杂度**$O(1)$：数组`counts`长度恒为`32`，占用常数大小的额外空间。
