# 题目描述

这是LeetCode上的[179.最大数](https://leetcode.cn/problems/largest-number/)。

难度：**中等**

Tag：贪心、数组、字符串、排序

给定一组非负整数 `nums`，重新排列每个数的顺序（每个数不可拆分）使之组成一个最大的整数。

**注意：**输出结果可能非常大，所以你需要返回一个字符串而不是整数。

示例1：

```
输入：nums = [10,2]
输出："210"
```


示例2：

```
输入：nums = [3,30,34,5,9]
输出："9534330"
```

提示：

- `1 <= nums.length <= 100`
- `0 <= nums[i] <= 10^9`

# 解题思路

此题求拼接起来的最大数字。设数组`nums`中任意两数字的字符串为`x`和`y`，则规定**贪心策略**：

- 若拼接字符串`x + y > y + x`，则认定`x`“**大于**”`y`。
- 反之，若`x + y < y + x`，则认定`x`“**小于**”`y`。

`x`“**小于**”`y`代表：排序完成后，数组中`x`应在`y`右边；“**大于**”则反之。

根据以上规则，套用任何排序方法对`nums`执行从大到小排序即可。

## 算法流程

- **初始化**：字符串列表`strs`，保存各数字的字符串格式。
- **列表排序**：根据贪心策略对`strs`进行从大到小排序。
- **返回值**：拼接`strs`中的所有字符串，并返回。

## 正确性证明

本题的贪心策略的正确性证明包括以下两个命题：

- **反身性**：对于任意的字符串数字`x`，有`xx = xx`。
- **传递性**：假设对于任意的字符串数字`x`，`y`，`z`，如果`xy < yx`，`yz < zy`，那么`xz < zx`一定成立。

反身性是显然成立的，因为任意数字和自己拼接总是相等。

下面证明传递性：

设十进制数`x`，`y`，``z`分别有`a`，`b`，`c`位，则有（左边是字符串拼接，右边是十进制数计算，两者等价）：
$$
\begin{cases}
xy = x \times 10^b + y \\
yx = y \times 10^a + x
\end{cases}
$$
则`xy < yx`可转化为： $x \times 10^b+y<y \times 10^a+x$

把含`x`的项移到一边，含`y`的项移到另一边： $x \times 10^b -x < y \times 10^a -y$

提取公因式： $x(10^b-1)<y(10^b-1)$

两边同时除以 $(10^a−1)(10^b−1)$，可得： $\frac{x}{10^a-1} < \frac{y}{10^b-1}$

同理，可得： $\frac{y}{10^b-1} < \frac{z}{10^c-1}$

合并传递可得： $\frac{x}{10^a-1} < \frac{z}{10^c-1}$，可逆推回拼接形式，即`z < zy`。

传递性证毕。

因此贪心策略是正确的，所得排序结果是全局最优的。

## 代码

内置函数方式：

```c++
class Solution {
public:
    string largestNumber(vector<int>& nums) {
        vector<string> strs;
        string res;
        for (int i = 0; i < nums.size(); i++)
            strs.push_back(to_string(nums[i]));
        // sort 的比较器：若 y+x < x+y 为真，则 x 应排在 y 之前，
        // 这样排序后，按顺序拼接得到的数字最大（降序排列）。
        sort(strs.begin(), strs.end(), [](string& x, string& y){ return y + x < x + y; });
        if (strs[0] == "0")
            return "0";
        for (int i = 0; i < strs.size(); i++)
            res.append(strs[i]);
        return res;
    }
};
```

快速排序：

```c++
class Solution {
public:
    string largestNumber(vector<int>& nums) {
        // 1. 将所有整数转为字符串
        vector<string> strs;
        for (int i = 0; i < nums.size(); i++)
            strs.push_back(to_string(nums[i]));

        // 2. 使用自定义快速排序，按“拼接后最大”的规则降序排列
        quickSort(strs, 0, strs.size() - 1);

        // 3. 处理全零的特殊情况：排序后最小的元素（最后一个）若是"0"，则所有元素均为0
        if (strs[strs.size() - 1] == "0")
            return "0";

        // 4. 按顺序拼接得到最大数
        // ⚠️ 注意：此处从后往前遍历是错误的！
        //    strs是降序排列（大的在前），正确做法应从前往后遍历：
        //    for (int i = 0; i < strs.size(); i++) res.append(strs[i]);
        //    当前写法会得到最小拼接数（升序），不符合题意。
        string res;
        for (int i = nums.size() - 1; i >= 0; i--)
            res.append(strs[i]);
        return res;
    }

private:
    /**
     * 快速排序（降序，按拼接后字典序比较）
     * 比较规则：若 a+b > b+a，则认为 a > b，a 应排在 b 前面。
     * 分区过程：将大于 pivot 的元素移到左边，小于 pivot 的移到右边。
     *
     * @param strs 待排序字符串数组（引用传递）
     * @param l    左边界索引
     * @param r    右边界索引
     */
    void quickSort(vector<string>& strs, int l, int r) {
        if (l >= r) return;

        int i = l, j = r;
        // 选取 strs[l] 作为基准 pivot
        while (i < j) {
            // 从右向左找第一个小于 pivot 的元素（应换到左边）
            while (strs[j] + strs[l] >= strs[l] + strs[j] && i < j) j--;
            // 从左向右找第一个大于 pivot 的元素（应换到右边）
            while (strs[i] + strs[l] <= strs[l] + strs[i] && i < j) i++;
            // 交换左边的一个“大元素”和右边的一个“小元素”
            swap(strs[i], strs[j]);
        }
        // 将 pivot 放到正确位置（i 处）
        swap(strs[i], strs[l]);

        // 递归处理左右子区间
        quickSort(strs, l, i - 1);
        quickSort(strs, i + 1, r);
    }
};
```

## 复杂度分析

- **时间复杂度**$O(N\log ⁡N)$：`N`为最终返回值的字符数量（`strs`列表的长度小于等于`N`）；使用快排或内置函数的平均时间复杂度为 $O(N\log⁡N)$，最差为 $O(N^2)$。
- **空间复杂度**$O(N)$：字符串列表`strs`占用线性大小的额外空间。
