# 题目描述

这是LeetCode上的[110.平衡二叉树](https://leetcode.cn/problems/balanced-binary-tree/)。

难度：**简单**

Tag：树、深度优先搜索、二叉树

给定一个二叉树，判断它是否是平衡二叉树。

**平衡二叉树**是指该树所有节点的左右子树的高度相差不超过1。

示例1：

```
        3
       / \
      9   20
         /  \
       15    7
输入：root = [3,9,20,null,null,15,7]
输出：true
```


示例2：

```
            1
           / \
          2   2
         / \
        3   3
       / \
      4   4
输入：root = [1,2,2,3,3,null,null,4,4]
输出：false
```

示例3：

```
输入：root = []
输出：true
```

提示：

- 树中的节点数在范围`[0, 5000]`内
- `-10000 <= Node.val <= 10000`

# 解题思路

当前树的深度等于左子树的高度与右子树的高度中的最大值加1。

定义函数`height`，用于计算二叉树中的任意一个节点`p`的高度：
$$
\text{height}(p) = 
\begin{cases} 
0 & p \text{ 是空节点} \\
\max(\text{height}(p.\text{left}), \text{height}(p.\text{right})) + 1 & p \text{ 是非空节点}
\end{cases}
$$

## 方法一：先序遍历+判断深度（从顶至底）

有了计算节点高度的函数，即可判断二叉树是否平衡。

具体做法类似于二叉树的前序遍历，即对于当前遍历到的节点。

首先计算左右子树的高度，如果左右子树的高度差是否不超过1，再分别递归地遍历左右子节点，并判断左子树和右子树是否平衡。

这是一个自顶向下的递归的过程。

### 算法流程

函数`height(root)`：计算树`root`的高度：

- **终止条件**：当`root`为空，即越过叶子节点，则返回高度0。
- **返回值**：返回左右子树高度的较大值与1的和。

函数`isBalanced(root)`：判断树`root`是否平衡

- **特例处理**：若树根节点`root`为空，则直接返回`true`。
- **返回值**：所有子树都需要满足平衡树性质，因此以下三个条件需要都满足才可以返回`true`
  - `abs(self.depth(root.left) - self.depth(root.right)) <= 1`：判断**当前子树**是否是平衡树。
  - `self.isBalanced(root.left)`：先序遍历递归，判断**当前子树的左子树**是否是平衡树。
  - `self.isBalanced(root.right)`：先序遍历递归，判断**当前子树的右子树**是否是平衡树。

### 代码

```c++
class Solution {
public:
    int height(TreeNode* root) {
        if (root == NULL) {
            return 0;
        } else {
            return max(height(root->left), height(root->right)) + 1;
        }
    }

    bool isBalanced(TreeNode* root) {
        if (root == NULL) {
            return true;
        } else {
            return abs(height(root->left) - height(root->right)) <= 1 
                    && isBalanced(root->left) 
                    && isBalanced(root->right);
        }
    }
};
```

### 复杂度分析

- **时间复杂度**$O(N\log N)$：其中`N`是二叉树中的节点个数。最坏情况下，二叉树是满二叉树，`isBalanced(root)`需要遍历二叉树中的所有节点，判断每个节点的高度`height(root)`需要遍历**各子树的所有节点**。
  - 满二叉树高度的复杂度 $O(\log N)$，将满二叉树按层分为 $\log_2(N+1)$层。
  - 通过调用`height(root)`，判断二叉树各层的节点的对应子树的高度。
    - 第一层有1个节点，需要遍历`N`个节点，因此该层遍历节点总数为 $\frac{N-0}{1}\times 1$。
    - 第二层有2个节点，每个节点遍历 $\frac{N-1}{2}$个节点，因此该层遍历总节点数为 $\frac{N-1}{2} \times 2$
    - 第三层有4个节点，每个节点遍历 $\frac{N-3}{4}$个节点，因此该层遍历总节点数为 $\frac{N-3}{4} \times 4$。
    - 第四层有8个节点，每个节点遍历 $\frac{N-7}{8}$个节点，因此该层遍历总节点数为 $\frac{N-7}{8} \times 8$。
    - 以此类推
    - 第 $\log(N+1)$层有 $2^{\log_2(N+1)-1}=\frac{N+1}{2}$个节点，每个节点遍历1个街道那，因此该层遍历总节点数目为 $1\times \frac{N+1}{2}$。
    - 因此各层执行`depth(root)`的时间复杂度为 $O(N)$（每层开始，最多遍历 $N$个节点，最少遍历 $\frac{N+1}{2}$个节点）。
  - 总体时间复杂度 $=$每层执行复杂度 $\times$层数复杂度 $=O(N \times \log N)$。
  - 对于最坏的情况，二叉树形成链式结构，高度为 $O(N)$，此时总时间复杂度为 $O(N^2)$。
- **空间复杂度**$O(N)$：最差情况下（树退化为链表时），系统递归需要使用 $O(N)$的栈空间。



## 方法二：后序遍历+剪枝（从底至顶）

方法一由于是自顶向下递归，因此对于同一个节点，函数`height()`会被重复调用，导致时间复杂度较高。如果使用自底向上的做法，则对于每个节点，函数`height()`只会被调用一次。

此方法为本题的最优解法，但剪枝的方法不易第一时间想到。

自底向上递归的做法类似于后序遍历，对于当前遍历到的节点，先递归地判断其左右子树是否平衡，再判断以当前节点为根的子树是否平衡。如果一棵子树是平衡的，则返回其高度（高度一定是非负整数），否则返回`−1`。如果存在一棵子树不平衡，则整个二叉树一定不平衡。

### 算法流程

函数`recur(root)`：

- **终止条件**：

  - 当`root`为空：说明越过叶节点，因此返回高度0。

  - 当左子树返回的高度为`−1`：代表左子树不是平衡树，因此剪枝，直接返回`−1`。
  - 当右子树返回的高度为`−1`：代表右子树不是平衡树，因此剪枝，直接返回`−1`。

- **返回值**：
  - 当前节点`root`的左右子树高度差≤1：则返回当前子树的高度，即节点`root`的左右子树高度的较大的较大值`+1`，即`max(left, right) + 1`。
  - 当前节点`root`左右子树的深度差>1：则返回`−1`，代表**此子树不是平衡树**。

函数`isBalanced(root)`：

- **返回值：**若`recur(root) != -1`，则说明此树平衡，返回`true`；否则返回`false`。

### 代码

```c++
class Solution {
public:
    bool isBalanced(TreeNode* root){
        return recur(root) != -1;
    }

    int recur(TreeNode* root){
        if (root == nullptr) return 0;
        int left = recur(root->left);
        if (left == -1) return -1;
        int right = recur(root->right);
        if (right == -1) return -1;
        return abs(left - right) < 2 ? max(left, right) + 1 : -1;
    }
};
```

### 复杂度分析

- **时间复杂度**$O(N)$：`N`为树的节点数；最差情况下，需要递归遍历树的所有节点。
- **空间复杂度**$O(N)$：最差情况下（树退化为链表时），系统递归需要使用 $O(N)$的栈空间。
