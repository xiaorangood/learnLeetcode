# 题目描述

这是LeetCode上的[10.正则表达式匹配](https://leetcode.cn/problems/regular-expression-matching/)。

难度：**困难**

Tag：递归、字符串、动态规划

给你一个字符串`s`和一个字符规律`p`，请你来实现一个支持`'.'`和`'*'`的正则表达式匹配。

- `'.'`匹配任意单个字符
- `'*'`匹配零个或多个前面的那一个元素

返回一个布尔值，表示匹配是否覆盖整个输入字符串（而非部分）。

示例1：

```
输入：s = "aa", p = "a"
输出：false
解释："a" 无法匹配 "aa" 整个字符串。
```


示例2：

```
输入：s = "aa", p = "a*"
输出：true
解释：因为 '*' 代表可以匹配零个或多个前面的那一个元素, 在这里前面的元素就是 'a'。因此，字符串 "aa" 可被视为 'a' 重复了一次。
```

示例3：

```
输入：s = "ab", p = ".*"
输出：true
解释：".*" 表示可匹配零个或多个（'*'）任意字符（'.'）。
```

提示：

- `1 <= s.length <= 20`
- `1 <= p.length <= 20`
- `s`只包含从`a-z`的小写字母。
- `p`只包含从`a-z`的小写字母，以及字符`.`和`*`。
- 保证每次出现字符`*`时，前面都匹配到有效的字符

# 解题思路

设`s`的长度为`n`，`p`的长度为`m`；将`s`的第`i`个字符记为 $s_i$，`p`的第`j`个字符记为 $p_j$，将`s`的前`i`个字符组成的子字符串记为`s[: i]`，同理将`p`的前`j`个字符组成的子字符串记为`p[: j]`。本题可转化为求`s[: n]`是否能和`p[: m]`匹配。

总体思路是从`s[: 1]`和`p[: 1]`开始判断是否能匹配，每轮添加一个字符并判断是否能匹配，直至添加完整个字符串`s`和`p`。展开来看，假设`s[: i]`与`p[: j]`可以匹配，那么下一状态有两种：

- 添加一个字符 $s_{i+1}$后是否能匹配？
- 添加字符 $p_{j+1}$后是否能匹配？

因此，本题的状态共有`m × n`种，应定义状态矩阵`dp`，`dp[i][j]`代表`s[: i]`与`p[: j]`是否可以匹配。

## 算法解析

做好状态定义，接下来就是根据“**普通字符**”、“**.**”、“**\***”三种字符的功能定义，分析出动态规划的转移方程。

**状态定义**：设动态规划矩阵`dp`，`dp[i][j]`代表字符串`s`的前`i`个字符和`p`的前`j`个字符能否匹配。

**转移方程**：需要注意，由于`dp[0][0]`代表的是空字符的状态，因此`dp[i][j]`对应的**添加字符**是`s[i - 1]`和`p[j - 1]`

> 例如`dp[3][3]`表示字符串`s`的前`3`个字符和`p`的前`3`个字符能否匹配，比较的添加字符是`s[0]`和`p[0]`。

- 当`p[j - 1] == '*'`时，`p[j - 2]*`是一个整体，表示`p[j - 2]`可以出现`0`次或多次。`dp[i][j]`在以下任一情况为`true`时等于`true`：
  - 当将字符组合`p[j - 2]*`看作出现`0`次时，
    - 查看`s`前`i`个字符与`p`前`j-2`个字符的匹配情况
    - 即`dp[i][j - 2]`
  - 当将字符组合`p[j - 2]*`看作`P[j-2]`多出现一次时，
    - 查看`s[i-1]`与`p[j-2]`的匹配情况（字符组合匹配这一次），
    - 且查看`s`前`i-1`与`p`的前`j`个字符的匹配情况（字符组合在之前已经匹配0次或多次）
    - 即`dp[i - 1][j] && s[i - 1] = p[j - 2]`
  - 当将字符`'.'`多出现一次时（`p[j - 2] = '.'`）
    - 查看`s`前`i-1`与`p`的前`j`个字符的匹配情况（字符组合在之前已经匹配0次或多次）
    - 即`dp[i - 1][j] && p[j - 2] = '.'`
- 当`p[j - 1] != '*'`时，`dp[i][j]`在当以下任一情况为`true`时等于`true`：
  - 字符`p[j - 1]`与字符`s[i - 1]`是否匹配：`dp[i - 1][j - 1]`且`s[i - 1] = p[j - 1]`
  - 字符`.`可以与字符`s[i - 1]`一定匹配，只要前面的字符匹配：`dp[i - 1][j - 1]`且`p[j - 1] = '.'`

**初始化**：需要先初始化`dp`矩阵首行，以避免状态转移时索引越界。

- **`dp[0][0] = true`**：代表两个空字符串能够匹配。
- **`dp[0][j] = dp[0][j - 2]`且`p[j - 1] = '*'`**：首行`s`为空字符串，因此当`p`的偶数位为`*`时才能够匹配（即让`p`的奇数位出现`0`次，保持`p`是空字符串）。因此，循环遍历字符串`p`，步长为`2`（即只看偶数位）。

**返回值**：`dp`矩阵右下角字符，代表字符串`s`和`p`能否匹配。

## 代码

```c++
class Solution {
public:
    bool isMatch(string s, string p) {
        // dp[i][j] 表示：s 的前 i 个字符 和 p 的前 j 个字符 是否匹配
        // 注意：这里的 i 和 j 是长度（从0开始），不是下标
        // 比如 s = "abc", dp[2][...] 代表 s 的前2个字符 "ab" 是否匹配
        int m = s.size() + 1; // +1 是因为要包含空字符串的状态 dp[0][...]
        int n = p.size() + 1;
        
        // 初始化 dp 表，全部设为 false
        vector<vector<bool>> dp(m, vector<bool>(n, false));
        
        // 空字符串和空模式是匹配的
        dp[0][0] = true;
        
        // ============ 初始化第一行：s为空字符串时的匹配情况 ============
        // s为空时，只有 p 能通过 * 消除前面字符变成空串，才能匹配
        // 例如 p = "a*b*c*" 可以变成空串，从而匹配空 s
        // 从 j=2 开始，是因为 * 至少需要前面有一个字符
        // j+=2 是因为 * 是成对出现的（字符+*），每次跳过这对
        for(int j = 2; j < n; j += 2) {
            // 如果 p[j-1] 是 *，那么能否匹配取决于跳过这 字符+* 组合 后是否匹配
            // 例如：p = "a*b*"，j=4 时(即处理到第二个 * )，
            // dp[0][4] = dp[0][2] && p[3]=='*'
            // 意思是：空串能否匹配 "a*b*"，取决于空串能否匹配 "a*" 且当前字符是 *
            dp[0][j] = dp[0][j - 2] && p[j - 1] == '*';
        }
        
        // ============ 状态转移 ============
        // i 遍历 s 的每个字符（1 到 m-1）
        // j 遍历 p 的每个字符（1 到 n-1）
        for(int i = 1; i < m; i++) {
            for(int j = 1; j < n; j++) {
                
                // ---- 情况一：p 的当前字符是 '*' ----
                // p[j-1] 是 *，它需要和前面的字符 p[j-2] 组合使用
                // 例如 p = "ab*"，p[j-2]='b'，p[j-1]='*'，组合为 "b*"
                if (p[j - 1] == '*') {
                    
                    // 子情况1：让 (p[j-2] + *) 匹配 0 次，也就是完全忽略这个组合
                    // 例如：s="a"  p="ab*"，忽略 "b*" 后变成 s="a" p="a"，这样可能匹配
                    // j-2 就是跳过这两个字符，回到这个组合之前的状态
                    if (dp[i][j - 2]) dp[i][j] = true;                      
                    
                    // 子情况2：让 (p[j-2] + *) 匹配 s 当前字符至少1次
                    // 前提1：s 少一个字符时，已经能用同一个模式 p 匹配（dp[i-1][j]）
                    // 前提2：s 的当前字符 和 p 的 * 前面的字符相同
                    // 详细解释：
                    //   dp[i-1][j] = true 表示：s去掉最后一个字符后，和整个p能匹配
                    //   s[i-1] == p[j-2] 表示：s的最后一个字符，正好能被*前面的字符匹配
                    //   既然之前就能匹配，现在末尾又来一个相同字符，* 就再吃一个
                    //   例如：s="aa" p="a*"
                    //     i=2,j=2: dp[1][2]=true(因为"a"和"a*"匹配) 且 s[1]='a'==p[0]='a'
                    //     所以 dp[2][2]=true ("aa"和"a*"匹配)
                    else if (dp[i - 1][j] && s[i - 1] == p[j - 2]) dp[i][j] = true;
                    
                    // 子情况3：和子情况2类似，但 p[j-2] 是万用符 '.'
                    // '.' 可以匹配任何字符，所以不需要判断字符相等
                    // 例如：s="aa" p="a*" 中的 * 前是 'a'，但如果 p=".*" 就能匹配任意字符
                    else if (dp[i - 1][j] && p[j - 2] == '.') dp[i][j] = true;
                    
                } 
                // ---- 情况二：p 的当前字符不是 '*'（普通字符或 '.'） ----
                else {
                    
                    // 子情况1：两个普通字符相同
                    // dp[i-1][j-1] 表示：在加这个字符之前，前面的部分都匹配
                    // s[i-1] == p[j-1] 表示：当前这两个字符也相同
                    // 例如：s="ab" p="ab"
                    //   i=2,j=2: dp[1][1]=true("a"=="a") 且 s[1]='b'==p[1]='b'
                    //   所以 dp[2][2]=true
                    if (dp[i - 1][j - 1] && s[i - 1] == p[j - 1]) dp[i][j] = true;
                    
                    // 子情况2：p 的当前字符是 '.'，可以匹配 s 的任何字符
                    // 前面的 dp[i-1][j-1] 保证了去掉这两个字符后，前面的部分匹配
                    // 例如：s="ab" p="a."
                    //   i=2,j=2: dp[1][1]=true("a"=="a") 且 p[1]='.' 万能匹配
                    //   所以 dp[2][2]=true
                    else if (dp[i - 1][j - 1] && p[j - 1] == '.') dp[i][j] = true;
                }
            }
        }
        
        // 返回整个字符串 s 和整个模式 p 是否匹配
        // dp[m-1][n-1] 就是 dp[s.size()][p.size()]
        return dp[m - 1][n - 1];
    }
};
```

双重循环内代码，即行列扫描，利用布尔运算实现简短长度可简化代码：

```c++
        for(int i = 1; i < m; i++) {
            for(int j = 1; j < n; j++) {
                dp[i][j] = p[j - 1] == '*' ?
                    dp[i][j - 2] || dp[i - 1][j] && (s[i - 1] == p[j - 2] || p[j - 2] == '.'):
                    dp[i - 1][j - 1] && (p[j - 1] == '.' || s[i - 1] == p[j - 1]);
            }
        }
```

## 复杂度分析

- **时间复杂度**$O(MN)$：其中`M`，`N`分别为`s`和`p`的长度，状态转移需遍历整个`dp`矩阵。
- **空间复杂度**$O(MN)$：状态矩阵`dp`使用 $O(MN)$的额外空间。
